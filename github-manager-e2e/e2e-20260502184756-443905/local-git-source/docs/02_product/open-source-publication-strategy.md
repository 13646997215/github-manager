# Open-Source Publication Strategy

## What makes this project competitive

ROS2-Agent should not compete as a generic AI wrapper.
It should compete by being more useful on real ROS2 engineering workflows.

## Competitive pillars

1. Strong workflow-specific skills
2. Structured diagnostics instead of vague explanation
3. Benchmarkable engineering tasks
4. Reproducible examples and broken cases
5. Hermes-native long-term memory and automation model
6. A path from solo developer usage to lab/team collaboration

## GitHub-facing priorities

- clear README message
- obvious repository structure
- tangible implemented assets, not only promises
- visible testing and validation
- reusable domain knowledge
- credible roadmap toward a full platform
