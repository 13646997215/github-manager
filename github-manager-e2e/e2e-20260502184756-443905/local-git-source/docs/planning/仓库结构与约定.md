# 仓库结构与约定

本文档定义 ROS2-Agent 仓库的骨架设计目标，确保项目从一开始就保持清晰、可扩展、适合 GitHub 发布。

## 目录目标

- `docs/`：所有架构、产品、路线图、工作流、规划文档
- `profile/`：Hermes profile 模板与上下文文件
- `skills/`：项目内 ROS2 领域技能资产
- `tools/`：结构化 Python 工具与规范
- `mcp/`：MCP 服务与连接器规划
- `examples/`：演示工作区、坏案例、样例 transcript
- `benchmarks/`：任务集、评分规则、基准报告
- `scripts/`：setup / audit / validation / cron 入口脚本
- `tests/`：技能、工具、工作流、文档一致性测试
- `assets/`：架构图、图片、可视化素材

## 命名原则

1. 文档优先使用英文路径，正文可中英结合。
2. 规划类文档可保留中文文件名，但尽量集中在 `docs/planning/`。
3. 新增资产必须放入明确分类目录，禁止散落在根目录。
4. 所有可执行脚本优先放在 `scripts/`。
5. 所有结构化 Python 能力优先放在 `tools/`。

## 开发规范

1. 每完成一个阶段，更新开发总日志。
2. 每个新增模块必须有用途说明。
3. 每个后续工具尽量有测试或样例输出。
4. 每个技能都要有适用场景、步骤、验证与风险说明。
